# Course Output Quality Loop

## Current Gap Audit

CourseForge already generated a Canvas-oriented shell with Start Here, content modules, final module, instructor module, rubrics, outcomes, assignment groups, Canvas navigation defaults, printable guide assets, IMSCC export, and import scaffolding. The remaining gap was instructional depth: generated content still leaned toward structured placeholders rather than a teachable first draft.

## Loop 1 Improvements

- Added structured course resources to the course model: textbook/OER/article/media placeholders, why-it-matters text, student instructions, instructor edit notes, estimated time, optional flags, and publish state.
- Added schedule entries to the course model for module release planning, resource work, practice, discussions, quizzes, and assignments. These remain date-less until term dates exist, avoiding fabricated due dates.
- Added a course quality scoring model with category scores, issues, recommended fixes, and auto-fix flags.
- Expanded module learning paths to include overview, readings/resources, substantive lesson content, practice activity, graded activities, and recap.
- Reworked lesson pages to include mini-lecture content, key terms, worked examples, misconception callouts, checks for understanding, and instructor teaching notes.
- Reworked assignments with scenario, task instructions, deliverables, format guidance, model elements, submission instructions, workload, rubric connection, and AI/integrity guidance.
- Reworked discussions with purpose, initial post instructions, reply expectations, facilitation tips, grading criteria, and example starter language.
- Expanded quiz question objects with difficulty, outcome alignment, module alignment, correct/incorrect feedback, and instructor review flags.
- Expanded syllabus output with required/optional materials, weekly schedule outline, late work, AI/integrity policy, technology requirements, help/support, accessibility, grading, outcomes, and workload sections.
- Added unpublished instructor module teaching notes with prep reminders, common student struggles, announcement ideas, grading guidance, and human review flags.
- Surfaced instructional quality scoring in the export/readiness area.

## Quality Categories Implemented

- Completeness
- Accessibility
- Outcome alignment
- Workload balance
- Assessment variety
- Instructor readiness
- Student clarity
- Canvas compatibility
- Syllabus quality
- Rubric quality
- Module learning path quality
- Export readiness

## Remaining Gaps For Later Loops

- Term-date settings, holiday/blackout handling, and actual due-date generation.
- Richer final project milestone variants by project type and course length.
- Resource block editing UI and uploaded-resource dependency export.
- Question banks larger than the active quiz question set.
- More detailed Canvas reimport conflict detection and selected-object export packaging.
- Human review checklist as a first-class generated object with must/recommended/optional sections.
- Accessibility validation beyond current HTML safety and media guidance checks.
- Canvas sandbox import verification against a real known-good export fixture.
- Full object-level revision coverage for resources, outcomes, modules, syllabus sections, instructor notes, and milestones.

## Verification

Current automated checks cover generated module structure, resource placeholders, quiz metadata, assignment groups, rubrics/outcomes, syllabus sync, IMSCC files, selected/changed export dependencies, internal link warnings, Canvas module import recovery, and representative quality thresholds for 4-, 6-, 8-, and 12-module course shapes.
